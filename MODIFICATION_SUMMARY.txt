1150713 系統專題審查報告－登入與部署修正版
完成日期：2026-07-22
依據：1150713_系統專題審查報告 (1)(1).docx

本次針對「此來源不允許呼叫 API」與登入無法使用完成：
1. 新增 lib/origin-policy.cjs，統一正規化 APP_BASE_URL、ALLOWED_ORIGINS 與 Render 網址。
2. CORS 會辨識 X-Forwarded-Host / X-Forwarded-Proto，與目前網站同來源的 API 不再被誤擋。
3. 保留白名單限制，未授權外部網站仍回傳 403。
4. 登入程式移至 public/login.js，使用 /api/... 相對路徑，避免前端拼錯 API 網址。
5. 新增 GET /api/login/status，登入頁可顯示 MongoDB 與 Email OTP 是否設定完成。
6. Email OTP 改支援通用 SMTP_*，並相容既有 Brevo、Gmail、EMAIL_* Render 環境變數。
7. 支援 users collection 中 password、passwordHash、password_hash 三種 bcrypt 欄位。
8. SMTP 寄送失敗時刪除未送達 OTP，避免等待重寄冷卻。
9. OTP 驗證限制為 6 位數，前端輸入框自動排除非數字。
10. login.html、login.js、sw.js 與 manifest 設定為不快取；Service Worker 更新為 aiot-static-v10。
11. 新增 CORS 同來源測試、登入結構測試與 GitHub Actions CI。
12. 新增 docs/LOGIN_TROUBLESHOOTING.md，更新部署、API 與審查要求完成狀態。
13. 清除 gitignore.txt、lib/validators.cjs、public/script.js 與舊修正文件等未使用檔案。
14. 加入 .npmrc，Render 固定由 https://registry.npmjs.org/ 安裝套件。

原審查報告既有完成項目仍保留：
- PWA 路徑與圖示、public/server.js 移除
- JWT/RBAC/tenant/system scope
- OTP MongoDB TTL、速率限制與密碼政策
- Helmet/CSP/HSTS、統一錯誤處理
- MQTT schema、E-stop ACK、AuditLog
- Summary aggregate、索引、CSV 匯出、Gemini fallback
